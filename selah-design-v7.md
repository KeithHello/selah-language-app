# Selah v7 設計文件

## 產品定位

Selah 是一個 iOS 獨佔的極簡語言學習 App。核心理念：**用戶不需要人指導就能使用**。

用戶的生活句子 → AI 翻譯成英文 → 成為學習材料。不是教科書，是用戶自己的人生。

目前支援英語，未來擴展日語。架構語言無關。

## 核心原則

1. **極簡 = 少於 5 個核心概念**：今日句子、聆聽、練習、我的筆記、精靈
2. **彈性優先**：不強制順序，根據時間和進度推薦下一步
3. **寵物 = 情緒鏡子**：一個維度（堅持程度），不是寵物模擬器
4. **iOS 原生**：Push navigation，Tab ≤ 2，利用 Widget 和 Haptic
5. **語言無關引擎**：四步流程不變，拆解邏輯按語言差異化
6. **錄音說中文**：用戶說中文，AI 翻譯成英文，英文成為學習材料

## 架構

### 2 Tab + Push Navigation

```
Tab 1：今日（首頁）
├── 精靈（情緒鏡子 + 今日小故事 + 動畫變化）
├── 時間感知問候
├── 智能推薦（與下方列去重：推薦聆聽時隱藏聆聽列）
├── 🎧 聆聽 → Push
├── ✏️ 練習 → Push
├── 🌙 夜間預覽（21:00+）→ Push
└── 🌙 今日一句 → Push

Tab 2：筆記
├── 我的句子（6 分類篩選，每分類至少 2 句）
├── 生詞列表（跨分類共用，附說明文字）
├── 搜尋
└── 🐦 小豆的回憶（寵物視角學習里程碑時間軸）
```

### 內容引擎

```
用戶錄音（中文）
  → STT 轉寫（保留不完美）
  → 教學化翻譯（自然英文 + 難度校準）
  → TTS 生成音頻（自然語調 + 速度控制）
  → 句子入庫，進入學習循環
```

### 句子生命週期

- Day 0：錄音產生 → AI 翻譯 → 入庫
- Day 0 晚：夜間預覽（首次看到英文）
- Day 1：聆聽練習（聽→猜→拆→說）
- Day 2-4：Quiz 練習（回憶）
- Day 7+：掌握或重新循環

## 聆聽流程（4 步）

1. **🎧 純聆聽**：閉眼聽 3 遍，訓練耳朵。TTS 預設 0.85x 速度，循環順序 0.85x → 1.0x → 1.2x → 0.7x。
2. **🧠 預測**：猜英文。最低 3 秒思考時間（太快則提示「再想想？」）。
3. **🔍 句子拆解**：揭示英文後才顯示拆解內容（與預測聯動隱藏）。標出核心詞組和句型結構。可點擊生詞查看詞義浮層。
4. **🗣️ 跟讀**：拆解完成後才顯示英文句子（不提前洩漏答案）。看著英文，開口說。錄音後自動播放原生音頻供比對。

## 練習流程（Quiz）

翻卡模式：看中文 → 試圖回憶英文 → 揭示 → 自評

三選自評：
- 「記得很清楚 ✅」
- 「差一點 🤔」
- 「完全不會 😵」

**規則：Quiz 只出已完成聆聽流程的句子。**

## 錄音流程

用戶說中文 → STT → AI 翻譯英文 → 難度校準 → 展示結果

STT 要求：
- 保留猶豫和自我修正
- 識別用戶自然使用的英文詞彙
- 清理純噪音但保留有語義的修正

翻譯要求：
- 自然口語，非教科書
- 有意識地稍微超前用戶水平
- 同一表達保持翻譯一致性

結果頁顯示：英文 + 中文 + 詞彙反饋（系統預圈候選詞 + 用戶勾選加入生詞本）

錄音頁提供 6 個話題方向（工作的事、朋友之間、想吐槽的、心裡話、我的想法、生活日常），選擇後顯示 4 個話題 starters 幫助用戶開始。

## 精靈系統

一個維度：堅持程度。

| 行為 | 精靈表現 |
|------|---------|
| 每天打開 | 活潑跳動，色彩飽和 |
| 完成聆聽 | 旁邊出現小音符 ♪ |
| 完成練習 | 開心旋轉 |
| 幾天沒來 | 安靜坐著，色彩稍淡 |
| 回來時 | 輕輕跳一下，「歡迎回來」 |
| 連續 7 天 | 外觀微變化（羽毛更亮） |

不該有的東西：XP、連續天數顯示、4 維度狀態條、成長階段系統。

### 精靈三層體驗設計

**第一層：小動作解鎖（新鮮感）**

精靈外觀（藍鳥）始終不變，但 idle 動畫隨 Day 數增加新動作。每個動作以低概率（~20%）隨機觸發，讓用戶每次打開今日頁面可能看到不同的精靈行為。

| Day 範圍 | 解鎖動畫 | 觸發概率 |
|----------|---------|---------|
| Day 1-3 | 基本漂浮（petFloat） | 100% |
| Day 4-6 | 歪頭（petTilt） | 20% |
| Day 7-9 | 轉圈（petSpin） | 20% |
| Day 10-13 | 拍翅膀（petFlap） | 20% |
| Day 14+ | 聆聽姿勢（petListen） | 20% |

設計原則：動作是「驚喜」而非「獎勵」。不顯示解鎖進度，不預告下一個動作是什麼。用戶偶然發現時會感到愉悅。

**第二層：今日小故事（情感連結）**

寵物下方顯示一行動態文字，根據用戶的學習行為和時間自動生成。內容分為 10 個類別，每類約 10 種變體，共 100 種故事。每次打開隨機選取一條符合當前條件的故事。

故事類別包括：新詞發現、錄音成就、聆聽進展、練習回憶、連續天數、時段感知、情緒觀察、趣味互動、學習洞察、里程碑預告。

示例：
- 「小豆今天學到了 iced latte，下次去咖啡店可以用！」
- 「連續 8 天了，小豆每天最期待的就是看到你 ♪」
- 「小豆覺得你今天特別認真，是不是有什麼目標？」
- 「如果小豆能喝咖啡，現在一定跟你一起喝 ☕」

**第三層：小豆的回憶（長期養成）**

筆記頁面底部的時間軸區塊，以寵物視角記錄學習里程碑。每個里程碑是一張卡片，包含日期、圖標、標題和一句寵物視角的描述。

里程碑類別包括：時間里程碑（Day 1/7/14/30/100...）、內容里程碑（第 10/25/50/100 句）、首次體驗（第一次猜對、第一次主動用詞）、特殊時刻（深夜學習、週末不斷更）、詞彙旅程（特定詞的使用軌跡）、成長觀察（句子變長、預測變準）。

設計原則：回憶是「內容的累積」而非「數字的增長」。不顯示 XP 條或等級數字，而是用具體的學習事件讓用戶感受到進步。共約 70 種里程碑模板。

## 生詞系統

生詞不是獨立功能，是句子學習流程的副產品。所有分類共用同一個生詞本，因為詞彙的生命週期天然跨越分類——一個詞可能在「社畜日常」的句子中首次遇見，後來在「朋友幹話」的錄音中被主動使用。

生命週期：遇見（預習點擊）→ 聽見（聆聽）→ 理解（拆解）→ 回憶（Quiz）→ 使用（錄音）

生詞本只是參考索引，每個詞帶著來源句子和使用記錄。

**生詞加入方式（兩步式）**：點擊高亮詞組時不直接加入，而是先彈出小浮層顯示詞義和用法說明，用戶確認後點擊「加入生詞本」才正式加入。這樣「看看意思」和「正式加入」是分開的操作。

**錄音結果的詞彙選擇**：AI 翻譯結果中預先圈出 3-5 個候選詞組，用戶勾選想加入的詞。系統預圈的不一定要選，用戶也可以點擊英文句子中的任何詞自行添加。

**生詞在學習中的角色**：Quiz 選題時優先包含生詞本中詞語的句子（自然間隔重複），但不單獨測試詞義。生詞的掌握通過「在錄音中主動使用」來判斷，而非選擇題。

用戶在錄音中主動使用某詞 = 該詞「已掌握」。

## 學習者引導（4 個干預）

1. **預測最低思考時間**：太快揭示時提示「再想想？」，再按一次直接揭示
2. **錄音後自動播放原生音頻**：讓用戶自然比對
3. **三選自評**：記得很清楚 / 差一點 / 完全不會
4. **學習教練**：前 5 次使用顯示步驟引導，之後自動消失

## 冷啟動策略

雙軌內容系統：
- 軌道 A（系統預置）：根據 Onboarding 種子句主題，預置 15-20 句
- 軌道 B（用戶生成）：從第一次錄音開始，逐漸接管

前 3 天 A 為主 → 第 4-7 天混合 → 第 8 天後 B 為主

## iOS 設計原則

- Tab Bar ≤ 2 個
- 子頁面用 Push navigation（非 Modal）
- Home Screen Widget：進度 + 精靈 + 打開按鈕
- Haptic Feedback：完成聆聽/通過 Quiz 時振動
- 時間感知：NSDate 判斷早晚，動態調整推薦
- 遵循 Apple HIG 間距和觸控目標（44pt）

## Onboarding 發音偏好

英語學習者在 Onboarding 第二步（語言選擇後）選擇發音偏好：

- 🇺🇸 美式發音
- 🇬🇧 英式發音

此偏好影響三個層面：
1. **TTS 聲線**：美式選 en-US，英式選 en-GB
2. **翻譯用詞**：AI 翻譯時傾向對應方言的詞彙（elevator vs lift, cookie vs biscuit）
3. **拆解說明**：遇到方言差異時主動提示（「英式說法：lift = 美式的 elevator」）

日語擴展時此步驟改為「標準語 / 關西弁」或隱藏。

## 前後端架構

### 分工原則

**能前端做的，絕不送後端。** 只有三件事必須後端做：聽（STT）、想（AI 翻譯）、說（TTS）。

### 五層架構

```
Layer 1: 用戶設備（iPhone）
  → 麥克風錄音、音頻播放、Haptic、Widget

Layer 2: 前端邏輯（App 內，零成本）
  → UI 渲染、時間感知、進度追蹤、寵物動畫、離線緩存、本地通知

Layer 3: API Gateway
  → 身份驗證、限流配額、請求路由

Layer 4: AI 服務層（產生費用的地方）
  → STT、AI 翻譯、TTS、詞彙分析、句子排程、寵物日記生成

Layer 5: 數據存儲
  → PostgreSQL（用戶數據）、S3/R2（音頻）、Redis（緩存）
```

### 前端負責（零成本）

- UI 渲染 + 所有動畫
- 時間感知智能推薦（只需 NSDate）
- 進度追蹤 + 間隔重複計算（本地優先）
- 寵物動畫選擇 + 今日小故事（本地數據庫隨機選取）
- 音頻播放控制（速度、暫停）
- Quiz 翻卡 + 三點自評
- 錄音 + 編碼（AVAudioRecorder）
- 本地通知排程
- Widget 渲染

### 後端負責（產生費用）

- STT 語音轉文字（Whisper / Deepgram）
- AI 翻譯引擎（GPT-4o-mini / Claude Haiku）
- TTS 文字轉語音（OpenAI TTS / ElevenLabs）
- 詞彙分析引擎（從翻譯結果中提取可教學詞組）
- 句子生命週期排程（Day 0 → Day 7+ 排程）
- 用戶數據同步（跨設備備份）
- 寵物日記生成（根據學習行為生成故事文案）

## 成本結構

### 每用戶每月成本估算

假設用戶每天錄音 1 次 + 聆聽 5 句 + Quiz 3 句。聆聽和 Quiz 幾乎零成本（音頻是預生成的，從本地緩存播放）。

| 環節 | 服務商 | 單次成本 | 月成本 | 佔比 |
|------|--------|---------|--------|------|
| STT | OpenAI Whisper | $0.001 | $0.03 | ~5% |
| AI 翻譯 | GPT-4o-mini | $0.0001 | $0.003 | ~1% |
| TTS | OpenAI TTS | $0.001-0.015 | $0.03-0.45 | **60-85%** |
| 基礎設施 | Supabase + R2 + Workers | — | $0-25 | ~10% |

**TTS 是最大成本項**，選錯服務商成本差 20 倍。

### 三檔成本方案

- **低成本 $0.05/月/用戶**：Whisper + GPT-4o-mini + OpenAI TTS
- **中等 $0.30/月/用戶**：Deepgram + Claude Haiku + ElevenLabs
- **高品質 $0.50/月/用戶**：Google STT + Claude Sonnet + ElevenLabs Pro

### 降本策略

1. **音頻緩存複用**：同一句英文 TTS 只生成一次，相似中文可能產出相同翻譯可複用（節省 30-50% TTS）
2. **批量預生成**：冷啟動句子庫提前批量生成翻譯+TTS，前 5 天零即時 API 成本
3. **非高峰生成**：用戶晚上錄音，TTS 深夜批量處理，次日可用
4. **用量分級**：免費用戶 1 次/天 + OpenAI TTS；付費用戶無限 + ElevenLabs

## 日語擴展預留

核心共用：句子為本、四步流程、精靈、筆記歸檔

差異化：
- 句子拆解：側重助詞 + 動詞變形 + 敬語層級
- 生詞類型：vocabulary / reading / grammar-particle
- 分類邏輯：加入禮貌程度維度（タメ口 / 敬語 / 謙讓語）
